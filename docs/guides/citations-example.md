# Citation Example

This page demonstrates how to use citations and bibliography in your documentation.

## What are Citations?

Citations are references to external sources within your documentation. They allow you to attribute ideas, data, and quotes to their original authors [citation:berners-lee_1991].

## Using REST APIs

The REST architectural style [citation:fielding_2000] provides a scalable approach to building web services. RESTful APIs have become the standard for web development.

## Data Format

JSON (JavaScript Object Notation) [citation:crockford_2001] is a lightweight data format that has become ubiquitous in modern APIs and web applications.

## Benefits of Citations

- **Credibility** — Show your sources and give credit
- **Academic Rigor** — Meet academic and professional standards
- **Traceability** — Readers can find the original sources
- **Professional** — Demonstrates thorough documentation

## How It Works

1. Define your bibliography in the frontmatter (YAML format)
2. Reference citations with `[citation:key_year]` in the markdown
3. The system automatically converts them to numbered references
4. Bibliography appears at the end with full details

## Multiple Authors

Citations can have multiple authors [citation:berners-lee_1991], which will be formatted correctly in the bibliography.
